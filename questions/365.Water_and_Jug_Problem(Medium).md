## 365. Water and Jug Problem (Medium)
**Date and Time:** Jun 6, 2026

Link: https://leetcode.com/problems/water-and-jug-problem/

<br>

### Question:
You are given two jugs with capacities `x` liters and `y` liters. You have an infinite water supply. Return whether the total amount of water in both jugs may reach `target` using the following operations:

* Fill either jug completely with water.
* Completely empty either jug.
* Pour water from one jug into another until the receiving jug is full, or the transferring jug is empty.

<br>

**Example 1:**
> **Input:** x = 3, y = 5, target = 4 <br>
> **Output:** true <br>
> **Explanation:** Follow these steps to reach a total of 4 liters: Fill the 5-liter jug (0, 5). Pour from the 5-liter into the 3-liter until full (3, 2). Empty the 3-liter jug (0, 2). Pour the 2 liters into the 3-liter jug (2, 0). Fill the 5-liter jug (2, 5). Pour from the 5-liter into the 3-liter until full (3, 4). The total is now 3 + 4 = 4 liters.

**Example 2:**
> **Input:** x = 2, y = 6, target = 5 <br>
> **Output:** false

**Example 3:**
> **Input:** x = 1, y = 2, target = 3 <br>
> **Output:** true

<br>

#### Constraints:
* `1 <= x, y, target <= 10^3`

<br>

### Walk-through: 
By **Bézout's identity**, any amount reachable by the two jugs must be a multiple of `gcd(x, y)`. So the problem reduces to two checks:

1. If `x + y < target`, it is impossible (not enough total capacity).

2. Otherwise, return `target % gcd(x, y) == 0`.

<br>

### Python Solution:
```python
class Solution:
    def canMeasureWater(self, x: int, y: int, target: int) -> bool:
        # Q: If both jugs can reach target
        # 1. Fill one jug full with water
        # 2. Empty one jug
        # 3. Pour water from one jug to another until receiving jug is full or transferring jug is empty
        # S: 1. Check gcd of x and y, if target is multiple of gcd, return True
        # 2. Edge case: if target > x + y, return False

        # TC: O(1), SC: O(1)
        if x + y < target:
            return False
        # Find common gcd
        def gcd(a, b):
            while b:
                a, b = b, a % b
            return a
        # 2. If target is multple of gcd, return True
        return target % gcd(x, y) == 0
```
**Time Complexity:** $O(\log(\min(x, y)))$, for the GCD computation via Euclidean algorithm. <br>
**Space Complexity:** $O(1)$, only constant extra space used.

<br>

### Python Solution (BFS):
```python
class Solution:
    def canMeasureWater(self, x: int, y: int, target: int) -> bool:
        # S: Two states a, b for x, y can have a + b = target
        # Run DFS on each state and update to check if can reach target
        # TC: O(x*y), SC: O(x*y)
        if x + y < target:
            return False
        # Check states
        visited = set()     # [jug1, jug2]
        deque = collections.deque()
        deque.append([0, 0])
        while deque:
            for _ in range(len(deque)):
                jug1, jug2 = deque.popleft()    # Current water for two jugs
                if jug1 + jug2 == target:
                    return True
                # Exclude visited states
                if (jug1, jug2) in visited:
                    continue
                visited.add((jug1, jug2))
                # Extend states for fill water, empty water
                deque.extend([[jug1, y], [x, jug2], [jug1, 0], [0, jug2]])
                # Extend states for transferring water
                # Transfer from jug1 -> jug2, min(left water in jug1, water can be added to jug2)
                water1 = min(jug1, y-jug2)
                deque.append([jug1-water1, jug2+water1])
                # Transfer from jug2 -> jug1, min(left water in jug2, water can be added to jug1)
                water2 = min(jug2, x-jug1)
                deque.append([jug1+water2, jug2-water2])
        return False
```
**Time Complexity:** $O(x \cdot y)$, the number of distinct `(jug1, jug2)` states is bounded by $(x+1)(y+1)$. <br>
**Space Complexity:** $O(x \cdot y)$, for the visited set and deque.
