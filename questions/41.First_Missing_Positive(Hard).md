## 41. First Missing Positive (Hard)
**Date and Time:** Jun 7, 2026

Link: https://leetcode.com/problems/first-missing-positive/

<br>

### Question:
Given an unsorted integer array `nums`, return the _smallest missing positive integer_.

You must implement an algorithm that runs in `O(n)` time and uses `O(1)` auxiliary space.

<br>

**Example 1:**
> **Input:** nums = [1,2,0] <br>
> **Output:** 3 <br>
> **Explanation:** The numbers in the range [1,2] are all in the array.

**Example 2:**
> **Input:** nums = [3,4,-1,1] <br>
> **Output:** 2 <br>
> **Explanation:** 1 is in the array but 2 is missing.

**Example 3:**
> **Input:** nums = [7,8,9,11,12] <br>
> **Output:** 1 <br>
> **Explanation:** The smallest positive integer 1 is missing.

<br>

#### Constraints:
* `1 <= nums.length <= 10^5`

* `-2^31 <= nums[i] <= 2^31 - 1`

<br>

### Walk-through: 
Use the array itself as a hash table via **in-place sign marking**.

1. **Clean up:** Replace any value that is `<= 0` or `> len(nums)` with `len(nums) + 1` (out-of-range placeholder that won't interfere).

2. **Mark:** For each value `n = abs(nums[i])`, if `1 <= n <= len(nums)`, negate `nums[n-1]` to record that integer `n` is present.

3. **Find:** The first index `i` where `nums[i] > 0` means `i+1` is missing. If all are negative, every integer `1..n` is present, so return `len(nums) + 1`.

<br>

### Python Solution:
```python
class Solution:
    def firstMissingPositive(self, nums: List[int]) -> int:
        # Q: Find the smallest positive integer missing in nums
        # S: In-place replacement to avoid O(n)
        # 1. clean up array: replace negative, 0, > len(nums) to be len(nums) + 1
        # 2. Mark nums[i] in nums[i] - 1 to be negative
        # 3. Traverse nums for each index i, if nums[i] > 0, this is the smallest element we missed. Otherwise, return len(nums) + 1
        # TC: O(n), SC: O(1)

        # 1. Clean up
        for i, n in enumerate(nums):
            if n <= 0 or n >= len(nums) + 1:
                nums[i] = len(nums) + 1
        # 2. Assign negative sign to valid n
        for i, n in enumerate(nums):
            # Convert n to abs(n) to avoid case if n is already negative
            n = abs(n)
            # Check if n is valid in [1, len(nums)], and not negative already
            if 1 <= n <= len(nums):
                nums[n-1] = -abs(nums[n-1])
        # 3. Check invalid index from nums
        for i, n in enumerate(nums):
            if nums[i] > 0:
                return i+1
        # Otherwise, [1, n] is valid, return n+1 to be the next missing positive
        return len(nums) + 1
```
**Time Complexity:** $O(n)$, `n = len(nums)`, three linear passes over the array. <br>
**Space Complexity:** $O(1)$, in-place modification with no extra data structures.
