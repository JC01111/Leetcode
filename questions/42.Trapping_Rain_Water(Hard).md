## 42. Trapping Rain Water (Hard)
**Date and Time:** Jul 8, 2024, 19:01 (EST)

Link: https://leetcode.com/problems/trapping-rain-water/

<br>

### Question:
Given `n` non-negative integers representing an elevation map where the width of each bar is `1`, compute how much water it can trap after raining.

<br>

**Example 1:**

<img src="../images/rainwatertrap.png" width=500>

> **Input:** height = [0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1]
> 
> **Output:** 6
>
> **Explanation:** The above elevation map (black section) is represented by array [0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1]. In this case, 6 units of rain water (blue section) are being trapped.

**Example 2:**
> **Input:** height = [4, 2, 0, 3, 2, 5]
> 
> **Output:** 9

<br>

### DP Solution:
Jun 6, 2026
```python
class Solution:
    def trap(self, height: List[int]) -> int:
        # Q: Find water we can trap
        # S: two dps, leftDp to save the maxLeft at index i, rightDp to save maxRight at index i
        # leftDp[i] = max(height[i], leftDp[i-1])
        # On the second pass, update res += min(leftDp[i], rightDp[i]) - height[i]
        
        # TC: O(n), n=len(height), SC: O(n)
        leftDp, rightDp = [0] * len(height), [0] * len(height)
        leftDp[0], rightDp[-1] = height[0], height[-1]
        res = 0
        # First pass to fill out leftDp
        for i in range(1, len(height)):
            leftDp[i] = max(leftDp[i-1], height[i])
        # Second pass to fill out rightDp
        for i in range(len(height)-2, -1, -1):
            rightDp[i] = max(rightDp[i+1], height[i])
            # Update res by counting leftDp
            res += min(leftDp[i], rightDp[i]) - height[i]
        return res
```
**Time Complexity:** $O(n)$ <br>
**Space Complexity:** $O(n)$

<br>

### Two Pointers Solution:
The two pointers solution is used to make sure the `l, r` pointers are the same, if they are not equal, we either increment `l` when `maxleft <= maxright` or decrement `r` when `maxleft > maxright`. Then we add `res` by either `maxleft - height[l]` or `maxright - height[r]`.

Notice that we update `maxleft = max(maxleft, height[l])`, so `res += maxleft - height[l]` will either be `0` (If `height[l+1]` updates `maxleft`) or the amount of water can be trapped in that index `i` (`maxleft` still larger than `height[l+1]`). Same for `maxright`.
```python
class Solution:
    def trap(self, height: List[int]) -> int:
        # S: two ptrs l, r to calculate # of water we can trap
        # maxLeft and maxRight to calculate difference between [l, maxleft] and [r, maxRight]
        # TC: O(n),n =len(height), SC: O(1)
        l, r = 0, len(height)-1
        maxLeft, maxRight = height[l], height[r]
        res = 0
        while l < r:
            # By doing this to make sure the min boundary is the same for maxLeft and maxRight
            if maxLeft <= maxRight:
                maxLeft = max(maxLeft, height[l])
                # If we update maxLeft, there will be no extra water. Otherwise, we can trap water
                res += maxLeft - height[l]
                l += 1
            else:
                maxRight = max(maxRight, height[r])
                # If we update maxRight, there will be no extra water. Otherwise, we can trap water
                res += maxRight - height[r]
                r -= 1
        return res
```
**Time Complexity:** $O(n)$ <br>
**Space Complexity:** $O(1)$
