## 1657. Determine if Two Strings Are Close (Medium)
**Date and Time:** Oct 21, 2024, 22:12 (EST)

Link: https://leetcode.com/problems/determine-if-two-strings-are-close/

<br>

### Question:
Two strings are considered **close** if you can attain one from the other using the following operations:

* Operation 1: Swap any two **existing** characters.
    * For example, a<u>b</u>cd<u>e</u> -> a<u>e</u>cd<u>b</u>

* Operation 2: Transform **every** occurrence of one **existing** character into another **existing** character, and do the same with the other character.
    * For example, <u>aa</u>c<u>abb</u> -> <u>bb</u>c<u>baa</u> (all `a`'s turn into `b`'s, and all `b`'s turn into `a`'s)

You can use the operations on either string as many times as necessary.

Given two strings, `word1` and `word2`, return `true` if `word1` and `word2` are **close**, and `false` otherwise.

<br>

**Example 1:**
> **Input:** word1 = "abc", word2 = "bca"
>
> **Output:** true
>
> **Explanation:** You can attain word2 from word1 in 2 operations. <br>
> Apply Operation 1: "a<u>bc</u>" -> "a<u>cb</u>" <br>
> Apply Operation 1: "<u>a</u>c<u>b</u>" -> "<u>b</u>c<u>a</u>"

**Example 2:**
> **Input:** word1 = "a", word2 = "aa"
>
> **Output:** false
>
> **Explanation:** It is impossible to attain word2 from word1, or vice versa, in any number of operations.

**Example 3:**
> **Input:** word1 = "cabbba", word2 = "abbccc"
>
> **Output:** true
>
> **Explanation:** You can attain word2 from word1 in 3 operations. <br>
> Apply Operation 1: "ca<u>b</u>bb<u>a</u>" -> "ca<u>a</u>bb<u>b</u>" <br>
> Apply Operation 2: "<u>c</u>aa<u>bbb</u>" -> "<u>b</u>aa<u>ccc</u>" <br>
> Apply Operation 2: "<u>baa</u>ccc" -> "<u>abb</u>ccc"

<br>

#### Constraints:
* `1 <= word1.length, word2.length <= 10^5`

* `word1` and `word2` contain only lowercase English letters.

<br>

### Walk-through:
We only need to compare the if two strings' keys are the same, and the total counts of values are the same as well.

<br>

### Python Solution:

```python
class Solution:
    def closeStrings(self, word1: str, word2: str) -> bool:
        # Two words are closed if another word can be done by operations
        # Sanity check, if lengths are not the same, return False
        # First of all, we know we want the values in hashmap to be the same for every key in each word
        # Otherwise, we are not able to do operation2.
        # Second, we want to make sure the cardinality between these words are the same as well.
        # Otherwise, the operation2 will not work again.

        # word1 = "abc", word2 = "cde"
        # hashmap1 = {"a": 1, "b":1, "c": 1}, hashmap2 = {"c": 1, "d":1, "e": 1}
        # See the values in hasmap1 and hashmap2 for each key can match up
        # Also number of keys in each hashmap can match up, so we can always do operation2.
        # Which means we always have a way to make them close

        # TC: O(n + m), n=len(word1), m=len(word2), SC: O(n+m)
        hashmap1, hashmap2 = {}, {}
        if len(word1) != len(word2):
            return False
        # add word into hashmap
        for i in word1:
            hashmap1[i] = hashmap1.get(i, 0) + 1
        for i in word2:
            hashmap2[i] = hashmap2.get(i, 0) + 1
        # Need to confirm we have the same cardinality for both words and same number of vals
        key1, key2 = sorted(hashmap1.keys()), sorted(hashmap2.keys())   # Time complexity
        val1, val2 = sorted(hashmap1.values()), sorted(hashmap2.values())
        return key1 == key2 and val1 == val2

        # Roughtly taken 48m
```
**Time Complexity:** $O(nlogn + mlogm)$, `n` is the length of `word1`, `m` is the length of `word2`. We sort both words.<br>
**Space Complexity:** $O(n + m)$
