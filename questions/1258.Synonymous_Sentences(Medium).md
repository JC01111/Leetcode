## 1258. Synonymous Sentences (Medium)
**Date and Time:** Jun 8, 2026

Link: https://leetcode.com/problems/synonymous-sentences/

<br>

### Question:
You are given a list of equivalent string pairs `synonyms` where `synonyms[i] = [s_i, t_i]` indicates that `s_i` and `t_i` are equivalent strings. You are also given a sentence `text`.

Return _all possible synonymous sentences **sorted lexicographically**_.

<br>

**Example 1:**
> **Input:** synonyms = [["happy","joy"],["sad","sorrow"],["joy","cheerful"]], text = "I am happy today but was sad yesterday" <br>
> **Output:** ["I am cheerful today but was sad yesterday","I am cheerful today but was sorrow yesterday","I am happy today but was sad yesterday","I am happy today but was sorrow yesterday","I am joy today but was sad yesterday","I am joy today but was sorrow yesterday"]

**Example 2:**
> **Input:** synonyms = [["happy","joy"],["cheerful","glad"]], text = "I am happy today but was glad yesterday" <br>
> **Output:** ["I am happy today but was cheerful yesterday","I am happy today but was glad yesterday","I am joy today but was cheerful yesterday","I am joy today but was glad yesterday"]

<br>

#### Constraints:
* `0 <= synonyms.length <= 10`

* `synonyms[i].length == 2`

* `1 <= synonyms[i][0].length, synonyms[i][1].length <= 10`

* `synonyms[i][0] != synonyms[i][1]`

* All words in `synonyms[i]` consist of lowercase English letters.

* `1 <= text.length <= 10^4`

* All words in `text` are separated by single spaces.

<br>

### Walk-through: 
Build an undirected **synonym graph**, then use **BFS** to find all words in each connected component, and **backtracking** to enumerate all sentence combinations.

1. Build `graph {word: [synonyms]}` from the synonym pairs (undirected).

2. For each word in `text`, run BFS to collect its entire connected component (all interchangeable words), then sort them lexicographically.

3. Run backtracking over the words in `text`, at each position trying every synonym from that word's component. At the leaf (all positions filled), join and append to `res`.

**TC note:** The user's original annotation `O(k^n)` is missing an O(n) factor for the `" ".join` at each leaf. Correct complexity is $O(n \cdot k^n)$.

<br>

### Python Solution:
```python
class Solution:
    def generateSentences(self, synonyms: List[List[str]], text: str) -> List[str]:
        # For each synonym in synonyms, build graph for them {word: [synonyms]}
        # Run backtrack to gather all the combination in lexicographically
        # Loop over text, if text[i] not in dict, append, until text[i] in dict, try all the combinations
        # TC: O(n * k^n), k=max size of synonym connected component, n=len(text), SC: O(n * k^n)
        
        res = []
        text = text.split()
        # 1. Add synonyms graph
        graph = collections.defaultdict(list)
        for s, t in synonyms:
            graph[s].append(t)
            graph[t].append(s)
        # 2. Loop over text by using BFS to get all synonyms
        def get_all_synonyms(word):
            if word not in graph:
                return [word]
            seen = set()
            queue = collections.deque([word])
            while queue:
                w = queue.popleft()
                if w not in seen:
                    seen.add(w)
                    for nei in graph[w]:
                        queue.append(nei)
            # Lexicographically order
            return sorted(seen)

        def backtrack(i, string):
            if i == len(text):
                res.append(" ".join(string))
                return
            for word in get_all_synonyms(text[i]):
                string.append(word)
                backtrack(i+1, string)
                string.pop()

        backtrack(0, [])
        return sorted(res)
```
**Time Complexity:** $O(n \cdot k^n)$, `n = len(text)`, `k` = max size of any synonym connected component; there are $k^n$ sentences and each takes $O(n)$ to build. <br>
**Space Complexity:** $O(n \cdot k^n)$, for storing all sentences in `res`.
