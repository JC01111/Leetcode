## 886. Possible Bipartition (Medium)
**Date and Time:** Jun 4, 2026

Link: https://leetcode.com/problems/possible-bipartition/

<br>

### Question:
We want to split a group of `n` people (labeled from `1` to `n`) into two groups of **any size**. Each person may dislike some other people, and they should not go into the same group.

Given the integer `n` and the array `dislikes` where `dislikes[i] = [a_i, b_i]` indicates that person `a_i` and person `b_i` dislike each other, return `true` _if it is possible to split everyone into two groups in this way_.

<br>

**Example 1:**
> **Input:** n = 4, dislikes = [[1,2],[1,3],[2,4]] <br>
> **Output:** true <br>
> **Explanation:** group1 [1,4] and group2 [2,3].

**Example 2:**
> **Input:** n = 3, dislikes = [[1,2],[1,3],[2,3]] <br>
> **Output:** false <br>
> **Explanation:** We need at least 3 groups to divide them.

**Example 3:**
> **Input:** n = 5, dislikes = [[1,2],[2,3],[3,4],[4,5],[1,5]] <br>
> **Output:** false

<br>

#### Constraints:
* `1 <= n <= 2000`

* `0 <= dislikes.length <= 10^4`

* `dislikes[i].length == 2`

* `1 <= a_i < b_i <= n`

* All the pairs of dislikes are **unique**.

<br>

### Walk-through: 
This is a **graph bipartiteness** check via DFS 2-coloring.

1. Build an undirected graph from `dislikes`.

2. Initialize `colors = [-1] * n` for all nodes uncolored.

3. For each uncolored node `i` (1 to n), run DFS starting with `color = 0`. In DFS:
   - Assign the current color to `node`.
   - For each neighbor, if it has the **same color** as `node`, return `False`.
   - If uncolored, recurse with the opposite color (`1 - color`).

4. If DFS completes without conflict for all components, return `True`.

<br>

### Python Solution:
```python
class Solution:
    def possibleBipartition(self, n: int, dislikes: List[List[int]]) -> bool:
        # Q: If we can separate all n ppl into two groups
        # S: 1. build undirected graph for a, b in dislikes[i]
        # 2. Start color with 0, then check its neighbors and color 1
        # 3. If neighbor's color the same as node, return False
        # TC: O(V+E), SC: O(V+E)
        # 1. Build graph
        graph = collections.defaultdict(list)
        for a, b in dislikes:
            graph[a].append(b)
            graph[b].append(a)
        # Check everyone from 1 to n
        colors = [-1] * n     # Initialize the color for everyone in -1
        def dfs(node, color):
            # Offset by -1 for i's color
            colors[node-1] = color
            # Check node's neighbors' colors
            for nei in graph[node]:
                # If nei has same color as node, it is not allowed
                if colors[nei-1] == color:
                    return False
                # If nei has no color, run DFS to color 
                if colors[nei-1] == -1 and not dfs(nei, 1-color):
                    return False
            return True

        for i in range(1, n+1):
            if colors[i-1] == -1:
                # If i has not been colored, color in 0
                if not dfs(i, 0):
                    return False
        return True

        # {1: [2,3], 2:[1,4], 3:[1]. 4:[2]}, 
        # colors = [0,1,-1,-1]
        # i=1, 2
```
**Time Complexity:** $O(V+E)$, `V = n`, `E = len(dislikes)`, each node and edge is visited once. <br>
**Space Complexity:** $O(V+E)$, for the adjacency list and recursion stack.
