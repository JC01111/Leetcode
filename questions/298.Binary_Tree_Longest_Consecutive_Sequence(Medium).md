## 298. Binary Tree Longest Consecutive Sequence (Medium)
**Date and Time:** Jun 9, 2026

Link: https://leetcode.com/problems/binary-tree-longest-consecutive-sequence/

<br>

### Question:
Given the `root` of a binary tree, return _the length of the longest consecutive sequence path_.

A **consecutive sequence path** is a path where the values **increase by one** along the path. Note that the path can go from parent to child only (cannot go from child to parent).

<br>

**Example 1:**
> **Input:** root = [1,null,3,2,4,null,null,null,5] <br>
> **Output:** 3 <br>
> **Explanation:** Longest consecutive sequence path is 3-4-5, so return 3.

**Example 2:**
> **Input:** root = [2,null,3,2,null,1] <br>
> **Output:** 2 <br>
> **Explanation:** Longest consecutive sequence path is 2-3, not 3-2-1, so return 2.

<br>

#### Constraints:
* The number of nodes in the tree is in the range `[1, 3 * 10^4]`.

* `-3 * 10^4 <= Node.val <= 3 * 10^4`

<br>

### Walk-through:
Run DFS top-down, passing `currCnt` (current consecutive length) and `prevVal` (parent's value) at each node.

1. If `node.val - prevVal == 1`, the sequence continues: `currCnt += 1`. Otherwise, reset `currCnt = 1`.

2. Update `res = max(res, currCnt)` at each node.

3. Recurse on both children, passing `node.val` as the new `prevVal`.

Initial call uses `prevVal = root.val`, which forces `currCnt = 1` at the root (since `root.val - root.val = 0 ≠ 1`), correctly treating the root as the start of a length-1 sequence.

<br>

### Python Solution:
```python
class Solution:
    def longestConsecutive(self, root: Optional[TreeNode]) -> int:
        # S: DFS and maintain currCnt, prev
        # TC: O(n), SC: O(1)
        res = 0
        def dfs(node, currCnt, prevVal):
            nonlocal res
            if not node:
                return
            if node.val - prevVal == 1:
                currCnt += 1
            else:
                currCnt = 1
            # Check left and right
            res = max(res, currCnt)
            dfs(node.left, currCnt, node.val)
            dfs(node.right, currCnt, node.val)
        
        dfs(root, 1, root.val)
        return res
```
**Time Complexity:** $O(n)$, every node is visited exactly once. <br>
**Space Complexity:** $O(h)$, `h` = height of the tree for the recursion call stack; $O(\log n)$ for a balanced tree, $O(n)$ worst case for a skewed tree.
