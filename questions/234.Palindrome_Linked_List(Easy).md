## 234. Palindrome Linked List (Easy)
**Date and Time:** Jun 7, 2026

Link: https://leetcode.com/problems/palindrome-linked-list/

<br>

### Question:
Given the `head` of a singly linked list, return `true` _if it is a palindrome or_ `false` _otherwise_.

<br>

**Example 1:**
> **Input:** head = [1,2,2,1] <br>
> **Output:** true

**Example 2:**
> **Input:** head = [1,2] <br>
> **Output:** false

<br>

#### Constraints:
* The number of nodes in the list is in the range `[1, 10^5]`.

* `0 <= Node.val <= 9`

<br>

### Walk-through: 
Use **fast/slow pointers** to find the midpoint, then **reverse the second half** in-place, then compare.

1. Advance `fast` two steps and `slow` one step until `fast.next` or `fast.next.next` is `None` — `slow` lands at the end of the first half.

2. Reverse the linked list starting from `slow`, producing a reversed second-half headed by `prev`.

3. Walk both `head` and `prev` forward together; if any values differ, return `False`. Otherwise return `True`.

<br>

### Python Solution:
```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution:
    def isPalindrome(self, head: Optional[ListNode]) -> bool:
        # Q: Return true if the linked list is a palindrome
        # S: 1. Run fast-slow ptr to find the middle node
        # 2. reverse the second half, compare the two halves to check palindrome. SC: O(1)
        # TC: O(n), SC: O(1)
        
        # 1. Run fast-slow ptr to find the middle node
        fast = slow = head
        while fast.next and fast.next.next:
            fast, slow = fast.next.next, slow.next
        # slow is the mid node
        # 2. Reverse the second half
        prev = None
        curr = slow
        while curr:
            currNext = curr.next
            curr.next = prev
            prev = curr
            curr = currNext
        # prev is now reversed second half head
        # 3. Check palindrome
        while head and prev:
            if head.val != prev.val:
                return False
            head, prev = head.next, prev.next
        return True
```
**Time Complexity:** $O(n)$, `n` is the number of nodes, two passes through the list. <br>
**Space Complexity:** $O(1)$, in-place reversal with no extra data structures.
