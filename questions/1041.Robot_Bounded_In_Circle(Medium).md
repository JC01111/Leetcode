## 1041. Robot Bounded In Circle (Medium)
**Date and Time:** Jun 9, 2026

Link: https://leetcode.com/problems/robot-bounded-in-circle/

<br>

### Question:
On an infinite plane, a robot initially stands at `(0, 0)` and faces north. Note that:

* The **north direction** is the positive direction of the Y-axis.
* The **south direction** is the negative direction of the Y-axis.
* The **east direction** is the positive direction of the X-axis.
* The **west direction** is the negative direction of the X-axis.

The robot can receive one of three instructions:

* `"G"`: go straight 1 unit.
* `"L"`: turn 90 degrees to the left (i.e., anti-clockwise direction).
* `"R"`: turn 90 degrees to the right (i.e., clockwise direction).

The robot performs the `instructions` given in order, and repeats them forever.

Return `true` _if and only if there exists a circle in the plane such that the robot never leaves the circle_.

<br>

**Example 1:**
> **Input:** instructions = "GGLLGG" <br>
> **Output:** true <br>
> **Explanation:** The robot is initially at (0, 0) facing north. After one cycle, it ends at (0, 0) facing north, so it stays bounded.

**Example 2:**
> **Input:** instructions = "GG" <br>
> **Output:** false <br>
> **Explanation:** The robot is initially at (0, 0) facing north. After one cycle, it ends at (0, 2) still facing north, so it moves away infinitely.

**Example 3:**
> **Input:** instructions = "GL" <br>
> **Output:** true <br>
> **Explanation:** The robot is initially at (0, 0) facing north. After one cycle, it ends at (0, 1) facing west. After two cycles, it ends at (-1, 1) facing south. After three cycles, it ends at (-1, 0) facing east. After four cycles, it ends at (0, 0) facing north. The robot is bounded.

<br>

#### Constraints:
* `1 <= instructions.length <= 100`

* `instructions[i]` is `'G'`, `'L'` or `'R'`.

<br>

### Walk-through:
Simulate one pass of the instructions. After one pass, the robot is bounded if:

1. It returns to `(0, 0)` — trivially bounded.
2. It does **not** face north (`index != 0`) — it will eventually cycle back:
   - Facing south: reverses direction each pass → returns after 2 passes.
   - Facing east or west: rotates 90° each pass → returns after 4 passes.

Use `directions = [[0,1],[1,0],[0,-1],[-1,0]]` (N, E, S, W). `L` decrements index by 1 mod 4 (or +3 mod 4); `R` increments index by 1 mod 4.

<br>

### Solution
Key point is if the robot back to origin (0, 0) or does not face North (0), we can always return the robot back to origin.
```python
class Solution:
    def isRobotBounded(self, instructions: str) -> bool:
        # Q: Return true/false if the robot can come back to origin
        # S: 1. Simulate all instructions
        # 2. Check final pos is the origin or not in the origin but facing not north/south
        # TC: O(n), n=len(instructions), SC: O(1)

        pos = [0, 0]
        direct = 0
        directions = [0, 1, 2, 3]   # [North, West, South, East]
        for c in instructions:
            if c == 'G':
                # Move 1 in direction
                if direct == 0:
                    pos = [pos[0], pos[1]+1]
                elif direct == 1:
                    pos = [pos[0]-1, pos[1]]
                elif direct == 2:
                    pos = [pos[0], pos[1]-1]
                elif direct == 3:
                    pos = [pos[0]+1, pos[1]]
            elif c == 'L':
                direct = (direct + 1) % 4
            elif c == 'R':
                direct = (direct + 3) % 4
        # Check pos and direct
        return True if pos == [0, 0] or direct != 0 else False
```
**Time Complexity:** $O(n)$, `n = len(instructions)`, single pass over instructions. <br>
**Space Complexity:** $O(1)$, only a fixed-size directions array and scalar variables.
